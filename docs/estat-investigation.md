# e-Stat 人口推計の調査結果

## 目的と選定方針

2015年以降の各年10月1日現在の「総人口・男女計」を、対象5都道府県について取得する統計表と分類コードをe-Stat API 3.0で確認した。

検索APIは候補発見だけに使用する。検索順位や公開日の新しさから統計表を自動選択せず、今回指定された次の2表を `getMetaInfo` で直接検証する。

- `0004021102`: 国勢調査結果による補間補正人口（2015年～2020年） 表005
- `0003448232`: 令和2年国勢調査基準 表005

平成27年より前の国勢調査基準表は、検索結果に表示しても「対象外」とする。平成27年基準など、それ以外の検索結果も自動選択せず「未検証」とする。

本調査では本番用の `population.json` を生成しない。

## 実行方法

`.env.example` を `.env` にコピーし、e-StatのアプリケーションIDを設定する。

```bash
npm run inspect:estat
```

アプリケーションIDはログ上でマスクされる。APIの生レスポンスは保存しない。

## APIリクエスト

実行時のリクエストは合計5回。

1. `getStatsList`: 候補発見
2. `getMetaInfo`: `0004021102`
3. `getMetaInfo`: `0003448232`
4. `getStatsData`: `0004021102`の2015年・2020年比較値
5. `getStatsData`: `0003448232`の2015年・2020年比較値

## 接続方針

APIメタ情報と重複年比較から、次工程の取得元候補は以下となる。

- 2015～2020年: `0004021102`（補間補正人口）
- 2021～2024年: `0003448232`（令和2年国勢調査基準）

2015年と2020年は両表で同値だった。境界を2020年と2021年の間に置けば、年の重複なしで接続できる。

ただし、`getMetaInfo` の解説情報から基準改定・不連続に関する注記は自動抽出できなかった。次工程では統計局の利用上の注意も確認し、系列接続の説明をデータのメタデータへ残す必要がある。
